Master Thesis Project

Micro-scale Particle Manipulation Using Bilateral Control

Technical Overview

Software

MATLAB, C++, OpenCV

Hardware

Motorized stage, Laser controller, High speed Camera, Linear Motor

Project Context

Optical tweezers use a highly focused laser beam to hold and move microscopic objects. For my thesis, I wanted to allow a human operator to physically feel the microscopic forces while manipulating these particles. I built a system that acts as a bridge between the macroscopic human world and the microscopic cellular world.

Development Process

First, I set up the hardware to read image data from the microscope camera. I used this visual information to track the position of the microparticles in real time.

Next, I calibrated the system using an experimental cross validation technique to measure the trap stiffness. This step was necessary to accurately calculate the invisible forces acting on the particle.

Finally, I implemented an admittance based bilateral control law. I used a reaction force observer to send tactile feedback to a linear motor. This allowed the user to move the linear motor by hand and feel physical resistance whenever the microscopic particle encountered an obstacle in the fluid.

Ecole Centrale de Nantes Research

Medical Cobotic Assistant

Technical Overview

Key Concepts

Impedance Control, Kinematics, Task Distribution, Manipulability Index

Software Tools

MATLAB, Simulink

Simulated Hardware

KUKA iiwa 7 R800 (7 DoF), RRR Serial Robot (3 DoF)

Project Context

I developed a simulation for the hand guidance of a macro micro redundant robot. The goal of this project was to assist in percutaneous medical interventions. Improving medical gestures leads to better patient care, reduces intervention costs, and improves the overall working conditions for practitioners.

The entire system was built as a simulation using MATLAB and Simulink. The setup consisted of a large 7 DoF KUKA robot acting as the macro system and a small 3 DoF robot acting as the micro system.

Development Process

First, I defined the configuration of the robot using Denavit Hartenberg parameters. A core requirement of the project was that the main KUKA arm had to be controlled by position while keeping a static orientation, whereas the smaller micro robot had to be controlled by torque.

I used a computed torque controller and implemented an impedance control law. This allowed the system to accurately simulate a physical force applied by the user when pushing the medical tool.

To make both robots move smoothly together, I implemented a task distribution algorithm. This logic calculates the manipulability index and the vector norm to dynamically adapt the gain between the macro and micro systems.

Project Overview

I built a complete ROS2 and Gazebo simulation of a robotic arm to detect and sort red and green apples. I used custom stereo vision and inverse kinematics to accomplish this task.

The custom control panel working together with the Gazebo environment and stereo vision cameras.

Software Architecture

For the vision system, I wrote a Python node using OpenCV. It finds circular shapes and calculates the depth distance using the visual difference between the left and right camera lenses. It then shares these 3D coordinates over the network.

For the arm movement, I created a C++ node. It receives the desired target coordinates and uses the Orocos Kinematics and Dynamics Library to calculate the exact joint angles needed to reach that position smoothly.

I also designed a control panel interface using Tkinter. It lets the user move the arm manually, read the transformation matrices in real time, and trigger automatic hovering and picking sequences.
