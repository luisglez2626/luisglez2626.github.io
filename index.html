Project ContextContexte du Projet

This project focuses on bridging the sensory gap in biological micromanipulation procedures, such as Intracytoplasmic Sperm Injection. Operators normally rely entirely on visual information to guess the forces applied to fragile cells. To solve this, a 2-axis bilateral teleoperation system was designed to pair a custom-built haptic master device with an optical tweezers slave manipulator. The system translates the macroscopic hand movements of the operator into precise microscopic actions, allowing the user to physically feel piconewton-scale interaction forces. Ce projet vise à combler le manque de retour sensoriel dans les procédures de micromanipulation biologique, telles que l'injection intracytoplasmique de spermatozoïdes (ICSI). Les opérateurs s'appuient normalement entièrement sur des informations visuelles pour estimer les forces appliquées aux cellules fragiles. Pour résoudre ce problème, un système de téléopération bilatérale à 2 axes a été conçu pour associer un dispositif maître haptique sur mesure à un manipulateur esclave (pinces optiques). Le système traduit les mouvements macroscopiques de la main de l'opérateur en actions microscopiques précises, permettant à l'utilisateur de ressentir physiquement les forces d'interaction à l'échelle du piconewton.

Development ArchitectureArchitecture de Développement

The hardware utilizes a Thorlabs CDL1015 continuous wave laser at 976 nanometers focused through a 100x oil-immersion objective to create the optical trap. The master interface features two linear motors equipped with high-resolution encoders. Le matériel utilise un laser à onde continue Thorlabs CDL1015 à 976 nanomètres, focalisé à travers un objectif à immersion dans l'huile 100x pour créer le piège optique. L'interface maître comprend deux moteurs linéaires équipés d'encodeurs haute résolution.

The software architecture relies on a C++ high-frequency control loop to manage an admittance control scheme. To provide force feedback without a dedicated physical sensor, the system incorporates a Reaction-Force Observer that estimates the interaction forces based on real-time tracking. A Disturbance Observer is also utilized to estimate and cancel out external mechanical disturbances on the linear motors, ensuring the operator only feels the true micro-environment forces. L'architecture logicielle s'appuie sur une boucle de commande haute fréquence en C++ pour gérer un schéma de commande en admittance. Pour fournir un retour de force sans capteur physique dédié, le système intègre un Observateur de force de réaction qui estime les forces d'interaction en temps réel. Un Observateur de Perturbations est également utilisé pour annuler les perturbations mécaniques externes sur les moteurs linéaires, garantissant que l'opérateur ne ressent que les véritables forces du micro-environnement.

For position tracking, a custom computer vision algorithm was implemented in Python using OpenCV. The software applies a Gaussian blur to reduce noise and uses thresholding to isolate the laser center. It then applies sub-pixel corner detection to accurately find the centroid of the trapped particle. This positional data is sent to the main controller via socket communication to calculate the exact forces. Pour le suivi de position, un algorithme de vision par ordinateur sur mesure a été implémenté en Python avec OpenCV. Le logiciel applique un flou gaussien pour réduire le bruit et utilise un seuillage pour isoler le centre du laser. Il applique ensuite une détection de coins sub-pixel pour trouver avec précision le centre de gravité de la particule piégée. Ces données sont envoyées au contrôleur principal via une communication par socket.

Accurate force estimation requires calculating the optical trap stiffness. The project evaluated several passive calibration techniques. The Equipartition, Boltzmann, and Mean Squared Displacement methods proved reliable. The Power Spectral Density analysis overestimated the stiffness because the camera's limited sampling rate of 107.9 Hz introduced motion blur and aliasing. L'estimation précise de la force nécessite le calcul de la rigidité du piège optique. Le projet a évalué plusieurs techniques d'étalonnage passif. Les méthodes d'équipartition, de Boltzmann et de déplacement quadratique moyen se sont révélées fiables. L'analyse de la densité spectrale de puissance a surestimé la rigidité en raison du taux d'échantillonnage limité de la caméra (107,9 Hz) qui a introduit un flou de mouvement.

Control & Kinematics StrategyStratégie de Commande et Cinématique

Robot configurations were initially defined utilizing Denavit-Hartenberg parameters to compute forward kinematics and Jacobian matrices. The manipulability index—a critical safety metric indicating proximity to mechanical singularities—was continuously derived from these matrices to ensure secure operations. Les configurations des robots ont été initialement définies à l'aide des paramètres de Denavit-Hartenberg pour calculer la cinématique directe (MGD) et les matrices Jacobiennes. L'indice de manipulabilité – une métrique de sécurité critique indiquant la proximité des singularités mécaniques – a été continuellement dérivé de ces matrices.

A core operational requirement mandated position control with a fixed orientation for the macro arm, while the micro robot operated exclusively via torque control. A computed torque controller featuring an impedance control law was engineered for the surgical tool. This configuration allows the mechanism to function as a virtual spring and damper, yielding safely to applied surgical forces and rendering the heavy macro robot virtually weightless during manual hand guidance. Une exigence opérationnelle fondamentale imposait une commande en position avec une orientation fixe pour le bras macro, tandis que le micro-robot fonctionnait exclusivement par commande en couple. Un contrôleur à couple calculé intégrant une loi de commande en impédance a été conçu pour l'outil chirurgical. Cette configuration permet au mécanisme de fonctionner comme un ressort et un amortisseur virtuels, rendant le lourd robot macro virtuellement en apesanteur pendant le guidage manuel.

Pour unifier les deux machines, un algorithme de distribution dynamique des tâches a été mis en œuvre. En surveillant l'extension physique du micro-robot (distance AB) et l'indice de manipulabilité actuel, l'algorithme évalue et dicte en temps réel si le bras macro doit assister le mouvement ou si le micro-outil dispose d'un espace de travail suffisant pour fonctionner de manière indépendante.

Implementation DetailsDétails d'Implémentation

This project represents a highly modular ROS 2 workspace designed for scalability, showcasing a complete migration to the modern Gazebo Harmonic (Ignition) simulation environment. The architecture strictly adheres to the principle of "separation of concerns," decoupling the kinematic logic, vision pipelines, hardware descriptions, and high-level applications into dedicated packages, efficiently managing transforms via `tf2` and hardware interfaces via `ros2_control`. Ce projet représente un espace de travail ROS 2 hautement modulaire conçu pour être évolutif, illustrant une migration complète vers l'environnement de simulation moderne Gazebo Harmonic (Ignition). L'architecture respecte strictement le principe de "séparation des préoccupations", en découplant la logique cinématique, les pipelines de vision, les descriptions matérielles et les applications de haut niveau dans des paquets dédiés, en gérant efficacement les transformations via `tf2` et les interfaces matérielles via `ros2_control`.

The core logic is driven by a comprehensive state machine (developed in Python) that acts as the orchestrator. It manages the full pick-and-place lifecycle: returning the arm to home, activating the vision system, commanding trajectory paths, toggling the custom vacuum gripper plugin, and depositing the sorted items. A custom-built Tkinter Graphical Interface runs concurrently, providing operators with a live dashboard. This UI displays real-time 6-axis joint states, live task progress, system alerts, and offers manual override controls over the state machine. La logique centrale est pilotée par une machine à états complète (développée en Python) qui agit comme orchestrateur. Elle gère l'ensemble du cycle de prise et dépose : retour du bras à la position d'origine, activation du système de vision, commande des trajectoires, basculement du plugin personnalisé du préhenseur à vide et dépôt des articles triés. Une Interface Graphique Tkinter sur mesure s'exécute simultanément, fournissant aux opérateurs un tableau de bord en direct. Cette interface affiche les états des 6 articulations en temps réel, la progression des tâches, les alertes du système, et offre des commandes d'annulation manuelle sur la machine à états.

For environmental perception, the vision node (collaborative_arm_vision) processes incoming stereo camera data. Using OpenCV, it isolates target geometries via HSV filtering, estimates physical depth using pixel disparity, and broadcasts these coordinates as dynamic 3D TF frames. Pour la perception de l'environnement, le nœud de vision (collaborative_arm_vision) traite les données entrantes de la caméra stéréo. En utilisant OpenCV, il isole les géométries cibles via un filtrage HSV, estime la profondeur physique en utilisant la disparité des pixels, et diffuse ces coordonnées sous forme de repères TF 3D dynamiques.

The kinematic solver (collaborative_arm_kinematics) serves as the bridge between perception and motion. Written in highly optimized C++ utilizing the Orocos Kinematics and Dynamics Library (KDL), it automatically parses the robot's URDF to build the kinematic tree. It listens for the TF frames published by the vision node and employs a Levenberg-Marquardt non-linear solver to calculate the exact inverse kinematics. It then packages these solutions into JointTrajectory messages for the Gazebo controllers to execute smoothly. Écrit en C++ hautement optimisé à l'aide de la bibliothèque Orocos KDL, il analyse automatiquement l'URDF du robot pour construire l'arbre cinématique. Il écoute les repères TF publiés par le nœud de vision et emploie un solveur non linéaire de Levenberg-Marquardt pour calculer la cinématique inverse exacte. Il emballe ensuite ces solutions dans des messages JointTrajectory pour que les contrôleurs Gazebo les exécutent de manière fluide.

Project DescriptionDescription du Projet

This project provides a robust, fully autonomous navigation and exploration stack designed for a 4-wheel independent swerve drive robot. Unlike standard differential drives, a swerve architecture offers exceptional maneuverability, allowing the robot to translate and rotate independently. The repository contains everything from custom kinematic controllers to high-level global path planning. Ce projet fournit une pile de navigation et d'exploration robuste et entièrement autonome conçue pour un robot à 4 roues motrices et directrices indépendantes (swerve). Contrairement aux entraînements différentiels standard, une architecture swerve offre une maniabilité exceptionnelle, permettant au robot de se translater et de tourner de manière indépendante. Le dépôt contient tout, des contrôleurs cinématiques personnalisés à la planification globale de trajectoire de haut niveau.

The core feature of this architecture is the implementation of a Rapidly-exploring Random Tree (RRT) algorithm. Instead of manually teleoperating the robot to map an unknown environment (like a maze), the RRT node automatically detects frontier regions—areas on the boundary between the known map and the unknown space. It continuously computes feasible target points and seamlessly dispatches Nav2 goal poses to autonomously generate a complete map using SLAM. La caractéristique principale de cette architecture est l'implémentation d'un algorithme RRT (Rapidly-exploring Random Tree). Au lieu de téléopérer manuellement le robot pour cartographier un environnement inconnu (comme un labyrinthe), le nœud RRT détecte automatiquement les zones frontalières — les zones à la limite entre la carte connue et l'espace inconnu. Il calcule en permanence des points cibles réalisables et envoie de manière fluide des objectifs Nav2 pour générer de manière autonome une carte complète via SLAM.
